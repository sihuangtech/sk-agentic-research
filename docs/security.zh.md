# 安全边界

## 当前保护

- 子进程参数列表执行，不使用 `shell=True`；
- 实验环境只保留必要系统变量，不传递模型、GitHub 等 API Key；
- Python AST 检查阻止动态执行、系统命令、递归删除和默认网络模块；
- Notebook 的所有代码单元执行前使用同一策略检查；
- 单文件最多 250 行；
- 实时监控超时、进程树内存和输出日志大小；
- 用户取消信号会终止当前进程组；
- 工作区路径解析阻止 `..` 和绝对路径逃逸；
- 默认需要人工审核实验计划和代码；
- Web 服务与 Docker Compose 默认只绑定 `127.0.0.1`。

## 这些保护不是什么

AST 规则不是强沙箱。Python 可以通过反射、第三方原生扩展、文件描述符或尚未覆盖的标准库路径绕过静态规则。本地执行器也不能提供内核级网络隔离。

因此：

- 不要在保存私钥、生产凭据或敏感数据的账号下运行不可信实验；
- 正式使用应实现 Docker/VM 执行器，并使用只读数据挂载、无网络、非 root 用户和 CPU/GPU 配额；
- 不要把 `allow_network: false` 理解为操作系统防火墙；
- 不要直接暴露 Web 控制台到公网，当前版本没有用户认证；
- LLM 生成代码必须经过人工审核，尤其是数据读取、模型下载和依赖要求。

## 依赖管理

核心依赖只在安装阶段通过 `python -m pip` 安装。运行时不会调用 `pip`、`apt-get` 或 `npm`。模型生成的实验也不允许自行安装依赖。需要新的实验库时，应由维护者审核后在独立实验镜像中通过官方包管理器安装。

Web 设置页可以更新 OpenAI、Anthropic 和 Google 的 API Key 与 Base URL。密钥只保存在权限收紧的本地 `.env` 中，读取接口仅返回是否已配置；空密钥表示保留原值，避免页面回显或意外覆盖密钥。
